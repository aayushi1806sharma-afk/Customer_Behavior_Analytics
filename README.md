# Customer Behavior Analytics Dashboard

## Overview

This project demonstrates an end-to-end Data Analytics workflow using Python, PostgreSQL, SQL, and Power BI. The objective was to analyze customer behavior data, extract meaningful business insights, and build an interactive dashboard to support data-driven decision-making.

The project covers the complete analytics pipeline, including data exploration, database integration, SQL analysis, and business intelligence reporting.

---

## Project Workflow

### 1. Exploratory Data Analysis (EDA)

* Imported and explored the customer behavior dataset using Python.
* Performed data cleaning and preprocessing.
* Analyzed customer demographics and behavioral patterns.
* Identified trends, distributions, and key insights through visualizations.

### 2. Database Integration

* Imported the cleaned dataset into PostgreSQL.
* Designed and managed database tables.
* Performed SQL-based analysis on customer data.

### 3. Business Analysis Using SQL

Executed multiple business-oriented SQL queries to answer questions such as:

* Which customer segments generate the highest revenue?
* What are the most common customer behaviors?
* Which products/categories perform best?
* How does customer engagement vary across segments?
* What are the key performance indicators (KPIs)?

### 4. Power BI Dashboard Development

* Connected Power BI directly to PostgreSQL.
* Built an interactive dashboard.
* Created visualizations for customer insights and business performance.
* Designed KPI cards and trend analysis reports.
